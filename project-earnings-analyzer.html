4 Agents. 60 Seconds. One Analyst Brief.

Analysts spend 2-4 hours per earnings call manually extracting metrics, reading tone, and writing briefs. I built a multi-agent AI system that does it in under a minute — and catches signals humans miss.

The Problem

Every quarter, thousands of publicly traded companies hold earnings calls. Each one is 60-90 minutes of corporate language packed with financial metrics, management tone signals, forward guidance, and analyst pushback. Professional analysts spend 2-4 hours per call extracting the signal from the noise.

The manual process has three failure modes. First, numbers get missed — a margin change buried in the CFO's prepared remarks, a segment revenue figure mentioned in passing. Second, sentiment is subjective — two analysts listening to the same call will disagree on whether the CEO sounded confident or evasive. Third, guidance tracking is tedious — comparing what management promised last quarter against what they actually delivered requires pulling up prior transcripts and manually cross-referencing.

What if an AI system could do all three simultaneously, in under a minute, and produce a structured analyst brief with a clear investment signal?

How It Works

The system uses a LangGraph state machine with four specialized agents. A shared state object flows through the pipeline, accumulating intelligence at each stage. Each agent reads from and writes to this state — downstream agents build on upstream analysis.

The Output: Three Signals

Every analysis ends with a decisive signal backed by the data. Not a hedge — a call.

BUY
Strong metrics beat, bullish sentiment, guidance raised, high management credibility
HOLD
Mixed signals, inline results, neutral sentiment, guidance maintained
WATCH
Missed expectations, cautious tone, guidance lowered, red flags detected

Why I Built It This Way

Why LangGraph, Not a Simple Chain

A simple prompt chain would lose structure between steps. LangGraph's typed state object means each agent's output is available to every downstream agent. The Sentiment Agent knows what the Metrics Agent found. The Report Agent sees everything. This is the difference between four independent analyses and one coherent intelligence product.

Why 4 Agents, Not 1 Mega-Prompt

One prompt trying to extract metrics AND analyze sentiment AND track guidance AND write a report produces mediocre results on all four. Specialized agents with narrow, well-defined tasks produce significantly better output. The Metrics Agent doesn't waste tokens on tone analysis. The Sentiment Agent doesn't get distracted by numbers. Focus beats breadth.

Why Sentiment Is a First-Class Signal

Most financial tools focus on numbers. But a CEO who says "we're cautiously optimistic" is telling you something different from one who says "we're confident in our trajectory." Hedging language, analyst deflections, restructuring vocabulary — these are leading indicators that numbers-only tools miss. The Sentiment Agent catches what spreadsheets can't.

Why Guidance Credibility Matters

A company that guides $5B and delivers $5.2B is signaling sandbagging. A company that guides $5B and delivers $4.5B is signaling execution problems. The Guidance Agent builds an accountability layer — tracking management's promise-vs-delivery pattern to weight the reliability of future guidance.

Tech Stack

Orchestration: LangGraph state machine — typed state flows through a directed acyclic graph of agent nodes. Each node enriches the shared state before passing to the next.

LLM: OpenAI GPT-4o — chosen for structured JSON extraction accuracy, low hallucination rate on financial data, and strong reasoning on sentiment nuance.

Ingestion: pdfplumber for PDF transcript parsing with automatic section detection (prepared remarks vs. Q&A) using regex pattern matching.

Lessons Learned

On agent specialization: The biggest accuracy gain came from splitting the mega-prompt into four focused agents. Each agent's system prompt is tuned for one job. The Metrics Agent is told "you are a precise financial data extraction agent" — not "you are a helpful assistant." Narrow instructions produce better output.

On structured outputs: Forcing every agent to return JSON instead of free text was a game-changer. It enables the dashboard visualization, makes cross-agent data flow reliable, and produces auditable output. If an agent can't fill a JSON field, it returns null — which is more honest than a hallucinated number in prose.

On the Q&A section: The most revealing part of an earnings call isn't the prepared remarks — it's the Q&A. When an analyst asks about margin pressure and the CFO pivots to talking about innovation, that's a signal. Auto-detecting the Q&A boundary and analyzing it separately from prepared remarks significantly improved sentiment accuracy.

On building for PMs, not quants: This tool isn't trying to replace a Bloomberg terminal. It's built for product leaders and business strategists who need to quickly understand a company's trajectory. The one-page brief format, the BUY/HOLD/WATCH signal, the plain-English sentiment read — these are designed for decision-makers, not traders.
